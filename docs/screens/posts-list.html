ActionResult
Tap post cardNavigate to Post Editor detail view
Long press post cardOpen context menu: Duplicate, Export to Album, Delete
Tap "+" buttonCreate new post (opens Post Editor with empty slots)
Tap filter tab (All/Draft/Ready/Published)Filter post list by status

Context menu: DeleteConfirmation dialog, then delete. Assets return to "non-utilise" state.
Edge Cases
  • No posts created: Empty state CTA "Create your first post" with illustration
  • Post with broken/deleted assets: Warning badge "1 broken" on post card, striped placeholder for missing slot
  • Cancel a post (delete): All assets return to "non-utilise" state and become available again
  • No swipe status change: Status can only be changed from within the Post Editor (intentional v0.1 decision)
  • Many posts (20+): Vertical scroll list, lazy loading
  • Auto-generated posts: Show "🎰 Auto" badge in purple next to status badge to distinguish from manually created posts
Acceptance Criteria
  • 4 visual slot thumbnails per post card (filled = thumbnail, empty = dotted outline)
  • Status badge on each card: Draft (gray), Ready (blue), Published (green)
  • "🎰 Auto" badge in purple shown on slot-machine-generated posts, next to status badge
  • Auto vs manual posts visually distinguishable at a glance
  • Tap post card navigates to Post Editor detail
  • Long press opens context menu with Duplicate, Export to Album, Delete
  • Delete returns all post assets to "non-utilise" state
  • Filter tabs (All, Draft, Ready, Published) work correctly
  • "+" button creates a new post
  • No swipe-to-change-status on post cards
