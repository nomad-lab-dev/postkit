Edge Cases
  • Delete pillar with assets: Assets lose that pillar tag but are NOT deleted. Confirmation shows "124 assets will lose this tag."
  • Duplicate pillar name: Inline error "A pillar with this name already exists." Save button disabled.
  • Empty pillar name: Don't allow save. Show inline error.
  • Last pillar deleted: Allowed. Home shows "Create your first pillar" empty state.
  • 11th pillar created: Nudge banner "Fewer pillars = stronger brand focus" (dismissible, non-blocking).
  • Very long pillar name (>30 chars): Max 30 chars enforced in text field
  • Emoji auto-suggest fails (no network): Default to star emoji
  • Reorder + iCloud sync (future): Last-write-wins for order field
  • Color picker on macOS vs iOS: Uses system ColorPicker (adaptive to platform)
Acceptance Criteria
  • Full CRUD operations work: create, read, update, delete pillars
  • Color picker renders correctly on both iOS and macOS
  • SF Symbol / emoji picker works with search functionality
  • Drag reorder persists pillar order
  • Delete removes pillar tag from assets (assets themselves are NOT deleted)
  • Delete confirmation dialog shows affected asset count
  • Duplicate pillar name prevented with inline error
  • 11+ pillars shows nudge banner (dismissible)
  • Empty name prevented from saving
  • Emoji auto-suggest on name field exit with micro-animation
