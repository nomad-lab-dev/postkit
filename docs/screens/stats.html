Edge Cases
  • No classified assets: All charts empty. Show "Scan your library to see stats" CTA pointing to Scan Settings.
  • Single pillar: Bar chart still renders with a single bar at 100%
  • No location data (EXIF stripped): "Top Locations" section shows explanation message instead of chart
  • Very skewed distribution (1 pillar has 90%): Chart scales appropriately, no overflow or visual glitch
  • Large library (15K+): Stats computed from SwiftData aggregates, not full table scans
Acceptance Criteria
  • Ring chart renders correctly showing classified vs unclassified percentage
  • Pillar distribution horizontal bar chart sorted by count descending
  • Framing distribution (POV, Detail, Wide, Portrait) renders as proportional blocks
  • Top 5 locations displayed from EXIF city data
  • Time of day distribution (Morning, Afternoon, Evening, Night) with mini bar charts
  • Meta stats show total assets, classified count, posts created, API calls, last scan info
  • All data is live from SwiftData queries
  • Tapping pillar bars and location rows navigates to Explore with filter applied
  • Empty state shown when no classified data exists
  • Pull to refresh recalculates stats
  • Charts adapt correctly to both iPhone and Mac layouts
