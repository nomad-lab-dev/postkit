Edge Cases
  • Invalid API key: "Test connection" fails with clear error message. Scan still works with Core ML fallback.
  • Cloud off + rescan: Uses Core ML only. Less accurate but free and fast. No API calls made.
  • Reset all classifications: Double confirmation required (dialog + type "RESET"). Irreversible operation.
  • API rate limit hit during scan: Pause scan, show error toast, offer to resume later with backoff
  • Keychain access denied: Show error explaining Keychain access is needed for secure key storage
  • Batch window larger than library history: Scans everything in one batch (no partial batching needed)
Acceptance Criteria
  • Cloud AI toggle works and visibly affects available options (API fields disabled when off)
  • API key stored securely in Keychain (not in SwiftData or UserDefaults)
  • "Test connection" validates API key with real API call and shows result
  • Batch settings (window + concurrency) persist and apply to next scan
  • "Scan new assets" starts incremental scan correctly
  • "Rescan all" requires single confirmation before proceeding
  • "Reset all classifications" requires double confirmation (dialog + type RESET)
  • Stats section displays correct numbers from SwiftData
  • API rate limit handling with pause + resume
