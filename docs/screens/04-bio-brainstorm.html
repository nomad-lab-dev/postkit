Edge Cases
  • Path A — empty bio submitted: Allowed. Persona marked as "No bio set."
  • Path A — over 280 chars: Counter turns red, field stops accepting input at limit
  • Path B — app killed mid-brainstorm: Answers saved per-question to ViewModel. Resume where left off on relaunch.
  • Path B — API error during AI reactions: Retry button. After 3 fails, fallback to 30 hardcoded questions (stored locally).
  • Path B — very short or empty answers: AI adapts and moves on. No judgment, no blocking.
  • Path B — user taps Back from brainstorm: Returns to path selector. All answers preserved in memory.
  • Path B — switch from brainstorm to Quick Bio: Allowed. Brainstorm answers preserved for later.
  • Synthesis — unsatisfactory result: User can edit inline or tap "Regenerate" for a new AI pass
  • Fallback questions: 30 hardcoded questions across 6 categories stored locally for offline use
Acceptance Criteria
  • Dual path selector renders with illustrations, playful copy, and tappable cards
  • Path cards centered with large emoji illustrations and inviting descriptions
  • Path A: bio field with playful placeholder text, 280-char counter, and "Let AI write it for me" helper
  • Path B: stacked cards with 2-card peek depth effect behind main card
  • Path B: multi-choice options + expandable free text per card
  • Path B: text field expand animation on tap (card grows smoothly)
  • Path B: AI reaction text appears between questions (from previous answer)
  • Path B: 6 categories visible as progress sections (Identity, Audience, Voice, Values, Content, Goals)
  • Path B: "That's enough" button appears after question 10
  • Path B: answers persist across app kills (saved per-question)
  • Path B: fallback to hardcoded questions on API failure
  • Synthesis: all 5 sections (Bio, Voice, Audience, Themes, Values) editable inline
  • Synthesis: "Regenerate" re-runs AI and replaces content
  • Fixed nav position consistent with all other onboarding steps
