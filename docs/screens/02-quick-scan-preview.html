Edge Cases
  • Permission denied : ecran lock + "Ouvrir les Reglages" deep link
  • Limited access : scan le subset, montre le resultat avec count limite + offre d'elargir
  • Galerie vide (0 photos) : message friendly + pas de CTA "next"
  • Core ML ne detecte rien de clair : "Pas de themes clairs" + skip vers pillars manuels
  • Que des screenshots : pilier "Tech/Screenshots" suggere, ou rien si trop generique
  • 20 photos toutes du meme sujet : 1 seul pilier suggere, c'est OK
Acceptance Criteria
  • Permission demandee contextuellement (pas dans un ecran dedie permissions)
  • Core ML analyse 20 photos recentes en moins de 10 secondes
  • Thumbnails apparaissent un par un avec animation fade-in
  • Piliers sugggeres avec thumbnails de SES photos (pas des placeholders)
  • Count de photos par pilier detecte
  • Permission denied/limited/empty geres avec UI appropriee
  • Les piliers sugggeres sont passes a l'etape 3 comme pre-selection
  • Step dots montrent 2/3
  • Nav buttons fixes en bas (coherent avec les autres etapes)
Design Notes
  • C'est l'ecran le plus important de l'onboarding — l'user voit la valeur sur SES photos en 10 sec
  • La permission est CONTEXTUELLE pas abstraite : "laisse-moi voir" vs "grant access"
  • Core ML + metadonnees EXIF (lieu, date) — pas de cloud API a ce stade
  • Les piliers sugggeres pre-remplissent l'etape 3 — l'user a moins de travail
  • Portfolio piece : VNClassifyImageRequest + PHAsset metadata
