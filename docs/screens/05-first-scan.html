Edge Cases
  • Permission already granted (re-install): Photos card pre-granted, Launch enabled immediately
  • Permission denied: red card + "Open Settings" deep link
  • Limited access: yellow card, scan subset or expand
  • No network: orange card, offer offline scan (Core ML only)
  • Empty library: grey card, "Take photos and come back"
  • Minimal setup (optional steps skipped, only pillars set): warning banner "No persona set — AI uses pillars only"
  • No skip/maybe later on this screen — only Back to return to previous step
  • App killed during first scan: resumes on next launch
  • Location permission NOT needed: EXIF GPS data read from photo metadata via PhotoKit
Acceptance Criteria
  • Persona recap centered with consistent alignment (all rows center-aligned)
  • Rocket icon + "Pre-launch checklist" header consistent across all states
  • 2 unlock cards only: Photo Library + Network (Location removed)
  • Cards: locked (grey/dashed/padlock) → granted (green/solid/emoji+data) with flip animation
  • Network auto-detected (no tap needed, green by default if online)
  • Launch button disabled until Photos card is granted
  • Launch button green when all granted
  • State 1 and State 2 have identical layout structure (only card states change)
  • State 3 transition screen with progress bar before auto-nav to Home
  • Edge cases: denied (red), limited (yellow), offline (orange), empty (grey)
  • Nav fixed position consistent with all onboarding steps
  • Step dots show 5/5
Design Notes
  • Location permission removed — EXIF GPS read from photo metadata, no system permission needed
  • Most users need only 1 tap (Photos) since Network is auto-granted
  • Card flip animation on unlock = satisfying micro-interaction (portfolio piece)
  • Consistent card layout: centered emoji on top, title, description — same in locked and granted
  • Variant chosen: Unlock Cards (B) + Mission Control rocket header (from brainstorm session)
  • No "Maybe Later" or "Skip" — permissions are mandatory, only Back to return to step 4
  • All strings in AppStrings enum for i18n readiness (no hardcoded text in Views)
  • Back button uses contextual text via NavigationStack (SwiftUI native)
