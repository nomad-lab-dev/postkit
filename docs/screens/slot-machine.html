Interactions
ActionResult
Wait during shuffleAnimation plays for 1-2s, then slots reveal one by one (top-left → top-right → bottom-left → bottom-right)
Tap "Shuffle 🎰"Re-shuffle all 4 slots with new photo combination (blur animation replays)
Tap "Editer ✏️"Navigate to Post Editor with these 4 photos pre-filled in slots
Tap "Garder ce post ✓"Save post as Draft, navigate to Post Editor
Tap "Garder quand meme" (partial)Save partial post, navigate to Post Editor with empty slots highlighted
Tap "Changer de template"Navigate to Template Builder to select a different template
Tap "Explorer"Navigate to Explore tab
Tap "Classifier"Navigate to Classification Queue
Edge Cases
  • Not enough classified assets for all slots: Partial fill with guidance on what cadrage/pillar photos are needed
  • All possible combinations already seen: Show message encouraging more classification to unlock new combos
  • No template configured: Redirect to template creation before slot machine can run
  • Same photo in multiple slots: Prevented by algorithm (each photo used once per post)
  • Shuffle hits same combination: Re-shuffle automatically until a new combination is found
  • App killed during shuffle animation: On relaunch, show revealed state with last generated combination
Acceptance Criteria
  • Shuffling state shows 4 blurred photos in 2x2 grid with progress indicator
  • Revealed state shows 4 clear photos with cadrage badges (POV, Detail, Wide, Portrait)
  • Blur-to-clear transition is visually satisfying (suggested by before/after states)
  • Template name displayed above the grid in revealed state
  • "Auto-generated" badge visible in top-right
  • Shuffle button triggers new combination (no duplicates with previous)
  • Edit opens Post Editor with photos pre-filled
  • Keep saves as Draft and navigates to Post Editor
  • Partial fill shows clear guidance for missing slot requirements
  • All-combinations-seen state provides clear next actions
