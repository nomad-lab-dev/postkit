Edge Cases
  • Permission denied : ecran lock + deep link "Ouvrir les Reglages"
  • Limited access : scan le subset, montre le resultat avec count limite + offre d'elargir
  • Galerie vide : message friendly + pas de CTA next
  • Core ML ne detecte rien : "Tes photos sont uniques" → skip vers pillars manuels
  • App killed pendant le scan : reprend au state 1 (le scan est rapide, pas grave)
  • 20 photos toutes du meme sujet : 1 seul pilier suggere, c'est OK
Acceptance Criteria
  • CTA "Decouvrir mon contenu" trigger la permission Photos
  • Permission → scan → resultats = meme ecran, 3 states animes
  • Core ML analyse 20 photos en moins de 10 secondes
  • Thumbnails apparaissent un par un (fade-in sequentiel)
  • Piliers sugggeres avec thumbnails de SES photos
  • Count de photos par pilier
  • Permission denied/limited/empty geres
  • Step dots montrent 1/2 (2 etapes d'onboarding total)
  • Les piliers sugggeres pre-remplissent l'etape 2
  • Un seul ecran SwiftUI avec 3 @State transitions
Design Notes
  • Fusionne Welcome + Quick Scan en 1 ecran (etait 2 ecrans separes en v0.4)
  • Onboarding = 2 etapes (Welcome/Scan + Pillars). Etait 3 en v0.4, 5 en v0.1, 7 en v0.0.
  • Le CTA trigger la permission de facon CONTEXTUELLE : "montre-moi mon contenu"
  • L'user ne voit jamais un ecran de permissions — c'est integre dans le flow naturel
  • Core ML VNClassifyImageRequest + PHAsset metadata (EXIF lieu/date)
  • Portfolio piece : 1 View SwiftUI, 3 states avec animations, permission handling
