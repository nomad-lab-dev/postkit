Interactions
ActionResult
Tap pillar cardNavigate to Explore tab, pre-filtered by that pillar
Tap "Scan Now"Start scan, card transitions to Scanning state with progress bar
Tap "Pause"Pause active scan, button changes to "Resume"
Tap "Review" (quick action)Navigate to Swipe tab (only visible when items to review exist)
Tap "Compose"Navigate to Posts tab, trigger new post creation flow
Tap "Create Template"Navigate to Template Builder screen
Tap template nudge "Creer →"Navigate to Template Builder screen
Pull to refreshCheck for new assets since last scan
Edge Cases
  • First launch (scan running): Scan card prominent at top with progress, pillar counts incrementing live, template nudge visible below scan card
  • No pillars (skipped onboarding): Show "Set up your brand" CTA instead of pillar grid
  • Zero assets in library: Empty state illustration, no pillar grid
  • Scan paused + app killed: Show "Resume scan?" prompt on relaunch
  • All classified, no posts yet: Nudge card "Ready to create your first post?"
  • New items detected: Yellow-bordered scan card with "Scan Now" CTA
  • 0 templates: Template nudge card visible with accent border. Disappears once first template is created.
  • No items to review: "Review" quick action hidden from Quick Actions grid
Acceptance Criteria
  • Scan status card reflects correct state among 4 variants (scanning, idle, new items, items to review)
  • Template nudge card visible when 0 templates (glass card with accent border + "Creer →" CTA)
  • Template nudge card disappears once first template is created
  • First launch state: scan running + pillars filling up incrementally + nudge visible
  • Review quick action only visible when items to review exist
  • Pillar bento cards show real thumbnails from SwiftData query (mini grid of 3-4 most recent)
  • Pillar cards show emoji, name, and asset count
  • Bento grid layout renders correctly (2 columns, no horizontal scroll)
  • Quick actions (Review, Compose, Create Template) navigate to correct destinations
  • Pull to refresh triggers new-asset check
  • 3-second rule: Home loads in under 3 seconds including thumbnails
