Interactions
ActionResult
Tap "Activer mes posts quotidiens"Navigate to Template Builder (pre-filled if preset selected)
Tap "Plus tard"Dismiss to Home Dashboard. Nudge card persists on dashboard until first template created.
Tap a preset card (Carousel, Story, Portrait+Hook)Visual selection highlight. "Creer mon template" opens builder pre-filled with that preset.
Tap on another presetSelection switches. Only one preset selectable at a time.
Edge Cases
  • "Plus tard" tapped: Home Dashboard shows nudge card with template CTA until first template is created
  • 0 pillars configured: Show fallback message suggesting pillar setup first
  • User already has a template: This screen should not appear (guard check in navigation)
  • No photos in library: Widget mockup still shows, but post preview uses placeholder images
  • App killed on this screen: On relaunch, go directly to Home Dashboard (onboarding considered complete)
Acceptance Criteria
  • iOS widget mockup renders with rounded corners, shadow, and system-like appearance
  • Pre-built post mockup uses real gallery photos with cadrage badges (POV, Detail, Wide, Portrait)
  • Template presets bento grid is selectable (single selection, visual highlight)
  • "Creer mon template" navigates to Template Builder, pre-filled if preset is selected
  • "Plus tard" dismisses to Home with nudge card persisted
  • No tab bar visible (transitional screen, not part of main app navigation)
  • No back button (post-onboarding only, forward flow)
  • Screen never shown again once a template is created
